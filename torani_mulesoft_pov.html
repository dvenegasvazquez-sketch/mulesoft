Section 03
MuleSoft Expansion Use Cases
P1 01 — TPM Data Foundation: Real-Time Trade Spend Intelligence Finance & Retail Commercial

The April TPM POV identified Finance, Retail Sales, and Demand Planning as the core functions driving the trade promotion management evaluation — and Torani's $1B revenue target requires those functions to make promotional investment decisions with the precision that scale demands. TPM platforms are only as effective as the data feeding them: retail sell-through rates, ERP accrual balances, distributor inventory positions, and promotional lift calculations must all be current and reliable for TPM to produce accurate ROI analysis. If Torani's TPM platform is connected to the same lagged SFTP data flows that Phase 1 is replacing, it will produce the same stale-data problem that afflicted 66% of the Tableau dashboards — but in a context where poor decisions cost margin at scale.

MuleSoft extends the Phase 1 integration foundation to connect the TPM platform directly to real-time data from retail scan feeds, ERP accrual systems, and distributor inventory APIs. A Process API normalizes promotional event data, accrual rates, and sell-through signals into the TPM platform's data model, enabling accurate pre-event planning, in-flight promotion monitoring, and post-event ROI reconciliation. The same API layer feeds Tableau and Data Cloud with the trade spend analytics that the Finance team needs for margin analysis and the Retail Sales team needs for account-level promotion planning.

For a company growing at 20%+ CAGR with an active retail expansion strategy, the ROI on trade spend accuracy compounds with revenue. A 1% improvement in promotional ROI across a $1B revenue base is $10M in margin impact. The TPM use case is the highest commercial-value expansion from Phase 1's foundation and the one most directly connected to Torani's stated growth ambition.

P1 02 — Distributor Data API: Real-Time Demand Sensing for the Flavor Factory Supply Chain & Demand Planning

Torani's Flavor Factory in San Leandro is a 327,000-square-foot production operation managing 400+ SKUs across both foodservice and retail channels. The production planning decisions that drive that facility's efficiency — which flavors to run, at what volumes, and when — depend on demand signals from the distributor network that currently arrive through the SFTP file processes Phase 1 is replacing. Even after Phase 1's go-live, the depth of real-time distributor data integration may be limited to the specific sources scoped in the initial contract. The full demand sensing capability — real-time POS data from foodservice distributors, retail scan data from grocery partners, and inventory position data from the distributor network — requires a more comprehensive API layer than Phase 1 alone delivers.

MuleSoft builds a Distributor Data API layer that connects the full range of Torani's distribution partners — foodservice distributors, retail DSD partners, and e-commerce fulfillment providers — through a normalized demand signal feed. A Process API aggregates these signals into a demand planning data stream that feeds both the demand planning system and Salesforce's commercial view of account performance. Flavor Factory production scheduling becomes demand-driven rather than forecast-driven, reducing overstock on slow-moving SKUs and preventing stockouts on high-velocity flavors like Vanilla and Caramel.

The Flavor of the Year cadence makes demand sensing acutely important: every new flavor launch requires Torani to predict demand accurately enough to avoid both excess inventory and distribution gaps during the launch window. Real-time distributor data makes that prediction meaningful rather than speculative — and feeds the cultural trend analysis that drives the next year's flavor innovation decisions with actual sell-through performance data.

P1 03 — Service Cloud Data Completeness: ERP and Order History Integration Customer Service & CRM

The +15 Service Cloud licenses closing in May represent Torani's investment in a customer service function that can serve its foodservice B2B accounts with the responsiveness that independent coffee shop and chain customers expect. But Service Cloud agents handling inquiries about order status, delivery timing, and account billing will face the same data completeness problem that affects every Service Cloud deployment without a connected ERP: the CRM has contact records and case history, but the operational data — order status, invoice history, shipment tracking, account credit standing — lives in the ERP and is not visible in Salesforce without an integration layer.

MuleSoft extends Phase 1's API foundation to connect Torani's ERP order and billing data to Service Cloud, surfacing real-time order status, invoice history, and account financial standing directly in the Service Cloud agent console. When a foodservice account manager from a regional coffee chain calls about a delayed shipment or a billing discrepancy, the agent sees the complete account picture — order history, open invoices, shipment status, and product mix — without leaving Salesforce. Agentforce (if Torani evaluates it in FY27-28) will depend on this same data completeness to automate routine service interactions.

The business outcome is a Service Cloud deployment that actually reduces handle time and improves first-call resolution from day one — rather than delivering a sophisticated UI with incomplete data behind it. For Torani's B2B foodservice customers, service quality is a retention lever, and a service organization that resolves inquiries without escalation builds the account loyalty that sustains the 20%+ CAGR growth trajectory.

P2 04 — Tableau Data Foundation: Governed API Feeds Replacing Stale SFTP Sources Analytics & Data Modernization

The October Tableau audit's finding — 66% of dashboards obsolete, only 35 critical — is a data freshness and completeness problem more than it is a Tableau configuration problem. Dashboards built on SFTP-delivered data batches become stale the moment the batch cycle ends; dashboards built on manually-maintained data sources become obsolete the moment someone stops maintaining them. Phase 1's SFTP replacement begins addressing this for the specific sources in scope, but the full Tableau portfolio renewal requires a systematic replacement of all SFTP-based data feeds with governed, real-time MuleSoft API connections.

MuleSoft's expansion use case here is architectural: every Tableau dashboard identified as "critical" in the October audit should have its underlying data source migrated from SFTP batch feeds to MuleSoft API connections as part of a phased analytics modernization program. New dashboards built on the API layer are reliable by construction — the data is current, the feeds are monitored through Anypoint Platform's Runtime Manager, and failures are alertable rather than silent. This creates the analytics foundation that Tableau optimization requires and that Salesforce Analytics Cloud expansion will depend on as Torani moves toward Data Cloud deployment.

The outcome is a Tableau estate that performs — 35 critical dashboards rebuilt on reliable real-time data, the remaining 227 obsolete workbooks retired rather than maintained, and new dashboard development governed by an API-first data sourcing standard that prevents the stale-data problem from recurring as Torani's analytics needs grow toward the $1B scale.

P3 05 — Data Cloud Foundation: MuleSoft as the Unified Data Ingestion Layer Data Cloud & Enterprise Strategy

Salesforce Data Cloud appeared in Torani's November SIC session as part of the 3–5 year enterprise data strategy. Data Cloud's value proposition — a unified customer data platform that enables AI-powered personalization, cross-channel identity resolution, and real-time segmentation — depends entirely on the quality and completeness of the data ingested into it. For Torani, that means unified profiles for its foodservice accounts that span order history, service interactions, promotional response, and product mix — data that currently lives across the ERP, Service Cloud, distributor feeds, and retail scan data in separate, unconnected systems.

MuleSoft is the correct and recommended data ingestion layer for Salesforce Data Cloud — a positioning Salesforce itself endorses. The API-led connectivity architecture established in Phase 1 and expanded through Phases 2 and 3 creates the normalized, governed data streams that Data Cloud requires for its unified profile model to be accurate and actionable. Building the Data Cloud architecture on MuleSoft-governed APIs rather than direct connectors means that when Torani's data sources change — a new distributor partner, an ERP upgrade, a new retail channel — the Data Cloud model is protected at the API boundary.

The business outcome is a Data Cloud deployment that, when Torani is ready to activate it, actually delivers the AI-driven commercial personalization and demand forecasting capabilities the investment promises — because the data foundation was built correctly from the start. Danielle's 3–5 year modernization vision ends with Data Cloud as the intelligence layer; MuleSoft is the infrastructure that makes that destination reachable.
